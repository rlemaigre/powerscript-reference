# Table 10.634:

|Argument|Description|
|---|---|
|key|A string whose value names the key in the system registry whose value you want to set.<br><br>To uniquely identify a key, specify the list of parent keys above it in the hierarchy, starting with the root key. The keys in the list are separated by backslashes.<br><br>If key does not exist in the registry, RegistrySet creates a new key. To create a key without a named value, specify an empty string for valuename.|
|valuename|A string containing the name of a value in the registry. Each key may have several named values. To specify the unnamed value, specify an empty string.<br><br>If valuename does not exist in the registry, RegistrySet causes a new name to be created for key.|
|valuetype|A value of the RegistryValueType enumerated datatype identifying the datatype of a value in the registry. Values are:<br><br>• RegString! -- A null-terminated string<br>• RegExpandString! -- A null-terminated string that contains unexpanded references to environment variables<br>• RegBinary! -- Binary data<br>• ReguLong! -- A 32-bit number<br>• ReguLongBigEndian! -- A 32-bit number<br>|



|Argument|Description|
|---|---|
| |• RegLongLong! -- A 64-bit number which is the longlong type ranging from 0 - 9,223,372,036,854,775,807, because the registry key value cannot accept a negative number.<br>• RegLink! -- A Unicode symbolic link<br>• RegMultiString! -- An unbounded array of strings<br>|
|value|A variable corresponding to the datatype of valuetype containing a value to be set in the registry.|
