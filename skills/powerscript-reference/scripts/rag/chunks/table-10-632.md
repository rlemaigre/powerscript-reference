# Table 10.632:

|Argument|Description|
|---|---|
|key|A string whose value names the key in the system registry whose value you want.<br><br>To uniquely identify a key, specify the list of parent keys above it in the hierarchy, starting with the root key. The keys in the list are separated by backslashes.|



|Argument|Description|
|---|---|
|valuename|A string containing the name of a value in the registry. Each key can have one unnamed value and several named values. For the unnamed value, specify an empty string.|
|valuetype|A value of the RegistryValueType enumerated datatype identifying the datatype of a value in the registry. Values are:<br><br>• RegString! -- A null-terminated string<br>• RegExpandString! -- A null-terminated string that contains unexpanded references to environment variables<br>• RegBinary! -- Binary data<br>• ReguLong! -- A 32-bit number<br>• ReguLongBigEndian! -- A 32-bit number<br>• RegLongLong! -- A 64-bit number which is the longlong type ranging from 0 - 9,223,372,036,854,775,807, because the registry key value cannot accept a negative number.<br>• RegLink! -- A Unicode symbolic link<br>• RegMultiString! -- An unbounded array of strings<br>|
|valuevariable|A variable corresponding to the datatype of valuetype in which you want to store the value obtained from the system registry for the specified key and value name.|
