# Table 10.488:

|Argument|Description|
|---|---|
|mailsession|A mailSession object identifying the session in which you want to read a message.|
|messageid|A string whose value is the ID of the mail message you want to read.|
|mailmessage|A mailMessage structure in which mailReadMessage stores the message information.|
|readoption|A value of the mailReadOption enumerated datatype:<br><br>• mailEntireMessage! -- Obtain header, text, and attachments<br>• mailEnvelopeOnly! -- Obtain header information only<br>• mailBodyAsFile! -- Obtain header, text, and attachments, and treat the message text as the first attachment, storing it in a temporary file<br>|



|Argument|Description|
|---|---|
| |mailBodyAsFile! is not supported for Extended MAPI. It works as mailEntireMessage! when used in PowerScript code.<br><br>• mailSuppressAttachments! -- Obtain header and text, but no attachments|
|mark|A boolean indicating whether you want to mark the message as read in the user's inbox. Values are:<br><br>• TRUE -- Mark the message as read<br>• FALSE -- Do not mark the message as read<br>|



Return value MailReturnCode. Returns one of the following values: mailReturnSuccess! mailReturnFailure! mailReturnInsufficientMemory! If any argument's value is null, mailReadMessage returns null. Usage To obtain the message IDs for the messages in the user's inbox, call mailGetMessages. Before calling mail functions, you must declare and create a mailSession object and call mailLogon to establish a mail session.
